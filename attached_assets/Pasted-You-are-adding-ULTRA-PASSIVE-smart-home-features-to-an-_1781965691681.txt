## GOAL
Add PassiveEco smart home integration + auto-bill-optimizer + auto-rewards to existing app WITHOUT breaking current functionality.

## DATABASE CHANGES (Add These Tables to Existing PostgreSQL)

Create these NEW tables:

1. smart_devices:
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   user_id UUID REFERENCES users(id) ON DELETE CASCADE,
   device_type TEXT NOT NULL, -- 'AC', 'light', 'plug', 'meter', 'tap'
   device_brand TEXT, -- 'Tuya', 'Philips Hue', 'Alexa'
   device_id TEXT NOT NULL, -- Unique device ID from brand
   device_name TEXT, -- User-friendly name
   is_connected BOOLEAN DEFAULT false,
   optimization_enabled BOOLEAN DEFAULT false,
   created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
   updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

2. auto_optimizations:
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   device_id UUID REFERENCES smart_devices(id) ON DELETE CASCADE,
   optimization_type TEXT NOT NULL, -- 'reduce', 'temp', 'off', 'cut_power'
   scheduled_time TIMESTAMP,
   actual_time TIMESTAMP,
   energy_saved_kwh DECIMAL DEFAULT 0,
   co2_saved_kg DECIMAL DEFAULT 0,
   money_saved DECIMAL DEFAULT 0,
   created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

3. eco_wallets (new coin system):
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   user_id UUID REFERENCES users(id) ON DELETE CASCADE,
   coin_balance DECIMAL DEFAULT 0, -- Eco Coins from passive savings
   money_balance DECIMAL DEFAULT 0, -- Cash from selling coins
   created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
   updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

4. eco_transactions:
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   user_id UUID REFERENCES users(id) ON DELETE CASCADE,
   transaction_type TEXT NOT NULL, -- 'earn', 'swap', 'sell', 'donate'
   coins DECIMAL NOT NULL,
   money DECIMAL DEFAULT 0,
   description TEXT,
   reference_id UUID, -- store_id, company_id, project_id
   created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

5. subscriptions:
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   user_id UUID REFERENCES users(id) ON DELETE CASCADE,
   bundle_type TEXT NOT NULL, -- 'eco_home_bundle'
   monthly_cost DECIMAL NOT NULL, -- ₹999
   coins_used DECIMAL DEFAULT 0,
   money_paid DECIMAL NOT NULL, -- ₹999
   status TEXT DEFAULT 'active', -- 'active', 'cancelled'
   started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
   cancelled_at TIMESTAMP

6. loyalty_levels:
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   user_id UUID REFERENCES users(id) ON DELETE CASCADE,
   level_name TEXT NOT NULL, -- 'Eco Beginner', 'Eco Smart', 'Eco Pro', 'Eco Legend'
   coins_required DECIMAL NOT NULL,
   rewards_monthly DECIMAL NOT NULL, -- ₹100, ₹300, ₹500, ₹1000
   achieved_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

7. neighborhood_stats:
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   neighborhood_name TEXT NOT NULL, -- 'T. Nagar, Chennai'
   total_homes INTEGER DEFAULT 0,
   total_co2_saved DECIMAL DEFAULT 0,
   total_money_saved DECIMAL DEFAULT 0,
   updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

Add indexes for: user_id in all tables, device_id in auto_optimizations.

## BACKEND API ENDPOINTS (Add to Your Express App)

Add these NEW routes to backend/src/routes.js:

1. POST /api/devices/connect - Connect smart device
   - Input: { device_type, device_brand, device_id, device_name }
   - Logic:
     • OAuth 1-tap with brand (Tuya/Philips Hue/Alexa)
     • Create smart_device record
     • Set is_connected = true
   - Return: { success: true, device_id, message: 'Device connected' }

2. GET /api/devices - List all user's devices
   - Return: smart_devices where user_id = req.user.id
   - Include: device_type, device_name, is_connected, optimization_enabled

3. POST /api/devices/optimize - Enable auto-optimization for device
   - Input: { device_id }
   - Logic:
     • Get device
     • Set optimization_enabled = true
     • Start AI learning (TensorFlow) for habit detection
     • Schedule auto-optimizations based on device type
   - Return: { success: true, message: 'Auto-optimization enabled' }

4. GET /api/savings - Get passive savings summary
   - Logic:
     • Sum auto_optimizations for user's devices
     • total_co2_kg = sum(co2_saved_kg)
     • total_money = sum(money_saved)
     • total_energy_kwh = sum(energy_saved_kwh)
   - Return: { co2_kg: 400, money: 750, energy_kwh: 1200 }

5. POST /api/savings/track - Auto-track optimization (called by devices)
   - Input: { device_id, optimization_type, energy_saved_kwh, co2_saved_kg, money_saved }
   - Logic:
     • Create auto_optimization record
     • Auto-earn Eco Coins: 1 coin per 0.01t CO₂ saved
     • coins_earned = co2_saved_kg / 10
     • Update eco_wallets.coin_balance += coins_earned
     • Create eco_transaction (type: 'earn', coins: +coins_earned)
   - Return: { success: true, coins_earned: 40, new_balance: 45 }

6. GET /api/wallet - Get user's eco wallet
   - Logic:
     • Get eco_wallets for user
     • If not exists, create with 0 balance
   - Return: { coin_balance: 45, money_balance: 180 }

7. POST /api/wallet/swap - Swap coins at store
   - Input: { deal_id }
   - Logic:
     • Get deal (coins_required, discount_value)
     • Check wallet.coin_balance >= coins_required
     • wallet.coin_balance -= coins_required
     • Create eco_transaction (type: 'swap', coins: -coins, money: +discount)
     • Generate QR code
   - Return: { success: true, discount: 150, qr_code: "QR_STRING" }

8. POST /api/wallet/sell - Sell coins to company
   - Input: { company_id, coins_to_sell }
   - Logic:
     • Get company offer (coins_per_unit, price_per_unit)
     • money_earned = (coins_to_sell / coins_per_unit) * price_per_unit
     • wallet.coin_balance -= coins_to_sell
     • wallet.money_balance += money_earned
     • Create eco_transaction (type: 'sell', coins: -coins, money: +money)
   - Return: { success: true, money_earned: 198, new_coin_balance: 0, new_money_balance: 378 }

9. POST /api/wallet/donate - Donate coins to project
   - Input: { project_id, coins_donated }
   - Logic:
     • Check wallet.coin_balance >= coins_donated
     • wallet.coin_balance -= coins_donated
     • Update project.raised_coins += coins_donated
     • Create eco_transaction (type: 'donate', coins: -coins)
     • Create project_donation record
   - Return: { success: true, new_balance: 35 }

10. POST /api/subscribe - Subscribe to Eco Bundle
    - Input: { bundle_type }
    - Logic:
      • Create subscription record (monthly_cost: 999, money_paid: 999, status: 'active')
      • Auto-enable optimization for all user's devices
      • Start AI habit learning
    - Return: { success: true, message: 'Subscribed to Eco Home Bundle', monthly_cost: 999 }

11. GET /api/loyalty - Get user's loyalty level
    - Logic:
      • Get user's wallet.coin_balance
      • Determine level:
        - Beginner: 0-100 coins
        - Smart: 100-300 coins
        - Pro: 300-500 coins
        - Legend: 500+ coins
      • Return level + rewards
    - Return: { level: 'Eco Beginner', rewards_monthly: 100, next_level: 'Eco Smart', progress: 45 }

12. GET /api/neighborhood - Get neighborhood stats
    - Input: neighborhood_name (from user profile)
    - Return: { neighborhood: 'T. Nagar, Chennai', total_homes: 45, total_co2_saved: 18000, total_money_saved: 27000 }
